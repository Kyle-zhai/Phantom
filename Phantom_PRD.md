## 01 产品概述

Phantom 是一款专注于「找回你的钱」的订阅管理工具。它不是预算 App，不是理财 App，只做一件事：

> *找出你正在付费但没有在用的所有订阅，让你用最少的操作取消它们。*

市面上最接近的竞品是 Rocket Money（原 Truebill），但它的核心商业模式是发放贷款和金融产品——订阅管理只是其获客漏斗。Phantom 做相反的事：**我们的商业模式就是帮用户省钱，而非向用户卖金融产品。**

### 1.1 问题的规模

订阅经济的增长制造了「订阅隐形债务」现象：

- Netflix、Spotify、Apple One、Amazon Prime、Hulu、Peacock、Adobe…… 平均美国用户有 12 个订阅
- 其中 4.5 个从未使用或极少使用，但仍在持续扣款
- 很多订阅以「免费试用」开始，试用期结束后自动转为付费，用户往往没有注意到
- 订阅涨价通知往往淹没在邮件里，用户不自知

> **用户原声（Reddit r/personalfinance 热门评论，点赞 8,400）**
>
> *「我今天审查了一下银行账单，发现自己还在付一个三年没用过的健身 App $19.99/月。三年就是 $720。这钱就这么消失了，我完全没注意到。」*

### 1.2 竞品分析与差异化

| 维度 | **Phantom ★** | Rocket Money | Truebill（已并购） | 银行 App 订阅视图 |
|------|-------------|-------------|-----------------|----------------|
| 核心定位 | 订阅侦探 + 退款助手 | 个人贷款 + 订阅工具 | 被 Rocket Money 收购 | 基础可视化 |
| 商业模式 | SaaS + 成功退款分成 | 贷款、信用建立产品 | 同 Rocket Money | 免费（银行自带） |
| AI 分析深度 | 「僵尸评分」算法 | 基础分类 | 基础分类 | 无 |
| 申诉信生成 | 一键生成正式信件 | 无 | 无 | 无 |
| 价格变动预警 | 7 天提前通知 | 无 | 无 | 无 |
| 数据安全承诺 | 不卖数据，不卖金融品 | 靠数据销售贷款 | 同上 | 银行内部 |
| 主要用户投诉 | —— | 「推销感太强」 | 「被卖给 Rocket Money 后变差了」 | 「太简陋」 |

> **核心差异化**：Rocket Money 的问题是利益冲突——它靠推销贷款赚钱，所以不会真心帮你省钱。Phantom 的商业模式与用户完全对齐——我们只在帮你省钱时才赚钱。这是产品哲学层面的根本区别，不是功能层面的微小差异。

## 03 核心功能设计

### 3.1 订阅雷达（Subscription Radar）

连接银行账户或信用卡后，Phantom 的 Radar 引擎做三件事：

- **识别**：扫描所有周期性扣款，覆盖 95% 的主流订阅服务
- **分类**：自动归类为「娱乐」「工具」「健康」「购物」「其他」
- **标注**：标记「免费试用即将结束」「上月开始」「连续 3 个月未使用（基于 App Store 活动数据）」

与竞品的关键差异：Phantom 不仅看扣款，还通过 Apple / Google 账号连接（可选）分析 App 最近启动时间，识别「付了钱但没用」的僵尸订阅。这是 Rocket Money 没有的数据维度。

### 3.2 僵尸评分算法（Zombie Score）

每个订阅都会获得一个 0–100 的「僵尸分」，分数越高越应该取消：

| 因素 | 权重 |
|------|------|
| 最近 30 天内 App 启动次数 | 35% |
| 相对于账单金额的使用时长 | 25% |
| 类似功能的重叠订阅 | 20% |
| 用户对该服务的主观评分 | 15% |
| 价格相对于市场均值 | 5% |

僵尸分 ≥ 80：系统主动推送「这个订阅值得重新考虑」通知，配合一键取消入口。

### 3.3 一键申诉信生成器（Dispute Letter Generator）

这是 Phantom **最独特的功能**，也是用户最愿意为之付费的功能。

针对「被收了本不应收的费用」的场景（如：忘记取消的试用期、自动续费未提前通知、已取消但仍然扣款），Phantom 一键生成专业的申诉信件，包括：

- 符合《电子资金转账法》（EFTA）条款的措辞
- 具体扣款日期、金额、账单参考号
- 礼貌但坚定的退款请求语气
- 如公司拒绝，升级至信用卡 chargeback 的指引

> **变现逻辑**：申诉信功能是 Phantom 最强的「成功退款分成」切入点——当用户通过申诉拿回 $200 退款时，Phantom 收取 15%（$30）。用户心甘情愿，因为他们本来会损失全部金额。

### 3.4 价格变动预警系统

Phantom 持续监控 2,000+ 主流订阅服务的价格变化，用户将在：

- **价格上涨前 7 天**收到推送通知
- 推送内附：当前价格 → 新价格 → 同类替代品推荐 → 一键取消链接

这一功能解决了一个关键用户痛点：「我根本不知道我的 Netflix 从 $15 涨到了 $22，就这样多付了 8 个月。」

### 3.5 订阅谈判助手

部分订阅服务（如 Hulu、Spotify、SiriusXM）在用户提出取消时会主动提供折扣。Phantom 系统化这一行为：

- 识别哪些服务有留存折扣策略（历史数据训练）
- 生成「请求留存优惠」的标准话术，适用于电话或在线聊天
- 追踪折扣是否兑现，到期后再次提醒

预计该功能可为每位活跃用户每年额外节省 **$80–150**。

## 04 商业模式

### 4.1 定价策略

| 层级 | 定价 / 内容 |
|------|-----------|
| 免费版 | 扫描最近 3 个月 / 识别最多 5 个订阅 / 基础分类 / 无 AI 分析 |
| Phantom Pro 月付 | $3.99/月 — 无限扫描 + 僵尸评分 + 价格预警 + 申诉信（5 份/月） |
| Phantom Pro 年付 | $29.99/年（节省 37%）— 同 Pro 月付 + 无限申诉信 + 优先支持 |
| 成功退款分成 | 15% 的申诉成功退款金额（仅用于用户主动触发的退款申诉） |
| 企业版 | $8/账号/月（5+ 账号）— 团队 SaaS 管理 + 统一报告 |

### 4.2 为什么定价如此低？

> **定价哲学：让省钱本身证明价值**
>
> Phantom Pro 年费 $29.99，相当于用户每月节省 $2.5 就回本。而我们的数据显示，平均用户通过 Phantom 每月节省 $47。ROI 是 **18.8 倍**。
>
> 低价不是为了获客——低价是因为我们想让每一个美国人都能用得起这个工具。规模化的用户基数才是商业价值。

## 05 数据安全与隐私

### 5.1 核心承诺：三不原则

> Phantom **不出售**用户数据给第三方。**不向**用户推销金融产品（贷款、信用卡）。**不存储**用户银行账号或信用卡卡号——我们只通过 Plaid API 获取只读的交易记录。
>
> 这三点是 Phantom 与竞品最根本的商业模式区别，也是我们的核心信任资产。

### 5.2 技术实现

- **银行连接**：通过 Plaid API（行业标准，Venmo / Coinbase 同款）获取只读交易数据
- **数据加密**：静态数据 AES-256，传输 TLS 1.3
- **最小化原则**：只拉取交易金额 + 商家名称 + 日期，不获取账号信息
- **用户完全控制**：可随时断开银行连接，数据删除请求 24 小时内响应
